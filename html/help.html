Adding projects (teams)

This is the first task to be completed. After deciding the number of teams, add one line per team in the Projects sheet. The only necessary data are:

  • the project's NAME (team's name) and
  • a unique KEY for the project. The key should contain no spaces.

Settings - Student emails

There are two options for students emails.

  • The best and simplest option is that students use their google domain emails for interacting with the PA system. In this case the registration of the student is a one-step process, in which students register with their names and project key (see below). To use this option make sure that in the Settings sheet the domain is set to true. Then use the menu PA -> Install -> Install Registration and Verification forms to create the forms for student's self registration.
  • If using google domain emails is not possible, then set the domain setting to false and then use the menu PA -> Install -> Install and Verification forms to create the forms for student's self registration. In this case, as explained below the registration is a two-step process including a verfication step after self-registration.

Student registration

Adding students manually

Students can be manually added in the Students sheet; add one line per student. The necessary data are:

  • First name,
  • last name,
  • email, and
  • project key (matching one of the keys in the Projects sheet).

The rest of the columns are filled by the system.

If you manually add students you have to notify them that they have to verify their accounts. Use the PA -> e-mails -> Send reminder to those who did not verify the account menu to let the system send the reminder for verification.

For google domain emails, you could skip the verification and registration steps altogether. Just enter the student data as described above and enter in the column VERIFIED the value TRUE for all the students that will be using their domain emails.

Students are added by self-registration

Get the Registration URL from the menu PA -> Links -> Registration URL by selecting and copying the displayed link (Ctrl-C). Announce to the students their project key for their project and the Registration URL.

After students fill in their details they will appear in the Students sheet.

Student account verification (only if not-domain emails have been used)

This step is not necessary if domain emails have been used for the registration of students. This is determined by the domain flag in settings.

All students (either added manually or via self-registration) have to verify their accounts. Verified accounts of students have TRUE in VERIFIED column.

You can use the PA -> e-mails -> Send reminder to those who did not verify the account menu to let the system send the reminder for verification to all students that have not verified their accounts yet.

Students will receive an email with their personal key and a link to the verification form in order to complete their registration to the system.

IMPORTANT: Students who don't verify their accounts can not participate in the peer assessment, although they will be evaluated by other students. This makes sure that their email addresses are correctly entered into the system. Check with students who don't verify their account, whether their email is correct.

Projects sheet

In the projects sheet you can see how many students belong to each group, how many have verified their account and how many have filled their peer assessment.

Peer assessments

Entering questions for the peer assessment

In the Questions sheet enter one row for each question of the peer assessment.

All questions will have an answer from 1 to 5 in the form.

Some sample questions are provided. Feel free to make any changes.

Entering peer assessments

Enter in the Peer Assessments sheet a row for the peer assessment with the following data:

  • Name of assessment
  • Unique KEY for the assessment
  • Deadline (make sure the date is correctly entered in the format YYYY-MM-DD hh:mm).

Opening the peer assessment

When you want to make the form available to students use the menu PA -> Peer assessments -> Open.

(Verified) Students will receive an email with the link in order to fill the peer assessment till the deadline.

Two more email reminders will be sent to students. One, 3 days before, and one 1 day before the deadline. The intervals of the reminders can be fully customized in the Settings sheet.

The form will automatically close at the deadline, and no more responses will be permitted, unless you change the deadline.

Changing the deadline of the peer assessment

You may change the date of the deadline in the Peer assessments sheet. You can do that either before or after the deadline has passed.

To complete the change use the menu PA -> Peer assessments -> Update deadlines.

Reminders will be sent again 3 and 1 days before the deadline.

Grading groups

In the sheet there is one line per assessment and project pair. In this sheet just enter the grade of the group in the specific assessment. The rest of the columns are already filled automatically. Necessary data:

  • Grade

Calculating peer assessment scores

Calculation

With the sheet active click on the row of one of the peer assessments. Then by using the menu PA -> Peer assessments -> Calculate you can have the system calculate individual scores.

A new sheet will be created including the [peer assessment id]. If the sheet already exists it will get cleared before it is filled.

Peer assessment Settings

The system uses the WEBPA algorithm for the peer assessment. You can change the peer assessment calculation settings in the Settings sheet and perform the calculation again. The parameters are:

  • PA Weight: the percentage of the grade that is adjusted.
  • PA penalty: the percentage of final mark reduction due to non-submission of peer assessment.
  • PA self: the peer assessment gather also self-assessment responses. A setting of true takes these into account, whereas a setting of false ignores the self-assessment responses.

Finalizing assessments

If you wish to announce the results to students select from the menu PA -> Peer assessments -> Finalize. A new sheet will be created with the name Final PA and the assessment's key. The sheet contains the results to be announced.

Finalized assessments should not change.

Announcing the results via email

PA -> Peer assessments -> eMail results will sent emails to all verified students. The email will contain the PA score and the final grade of the peer assessment. You can set at the Settings sheet whether you wish one or both of them not sent.
